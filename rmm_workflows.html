1

Scheduled Inspection Workflow

From scheduling through completion — and how findings become tasks

flowchart TD A([📅 Inspection Scheduled]) --> B[Template Assigned\ne.g. Food Safety Routine] B --> C[/Appears on Owner\n+ Operator Dashboards/] C --> D[Inspector Opens Inspection] D --> E[Walk Through Section] E --> F{Item Result} F -- PASS --> G([✓ Next Item]) F -- FAIL / FLAG --> H[Create Task from Item] G --> E H --> I[(Task Created\nwith context pre-filled)] E -- All sections done --> J[Complete Inspection] J --> K[/Inspection Report\nScore + Findings/] K --> L([📋 Submitted & Archived]) I --> M([Task Lifecycle ↓ see Diagram 2]) style A fill:#dbeafe,stroke:#93c5fd,color:#1e40af style L fill:#dcfce7,stroke:#86efac,color:#166534 style M fill:#fef3c7,stroke:#fcd34d,color:#92400e style H fill:#fee2e2,stroke:#fca5a5,color:#991b1b style I fill:#fee2e2,stroke:#fca5a5,color:#991b1b

Key point: An inspection is a point-in-time event with a start and end. It produces a score and may produce zero or many tasks. Templates define what gets checked — they live at the location level and can be customised per restaurant type.

2

Task Lifecycle

A task stays open until explicitly resolved — it's a living record

flowchart TD A([Task Created]) --> B[Assigned to\nInspector or Operator] B --> C[/Task Detail — OPEN/] C --> D[Add Evidence\nPhotos · Audio · Notes] C --> E[Add Update\nLog progress] D --> C E --> C C --> F{Resolution Action} F -- Mark Resolved --> G([✓ Task Closed\nTimestamp recorded]) F -- Request Reinspection --> H([New Inspection\nScheduled]) F -- Escalate Issue --> I([Critical Alert\nFlagged to Owner]) G --> J[/Archived Record\nRead-only history/] H --> K([Back to Diagram 1]) I --> L[Owner Reviews\n+ Assigns follow-up] L --> C style A fill:#dbeafe,stroke:#93c5fd,color:#1e40af style G fill:#dcfce7,stroke:#86efac,color:#166534 style J fill:#f3f4f6,stroke:#d1d5db,color:#374151 style I fill:#fee2e2,stroke:#fca5a5,color:#991b1b

Key point: A task is never deleted — only resolved and archived. Every update, photo, and note is appended to the timeline. The record shows the full history from creation to resolution.

3

Triggered Task Creation

Tasks created outside of a scheduled inspection — reactive workflow

flowchart TD A{Trigger Source} --> B[Inspection Finding\nduring walkthrough] A --> C[Incident Report\nmanual entry via New Risk] A --> D[Failed Public\nHealth Inspection] A --> E[Internal Score\nbelow threshold] B --> F[/Task Auto-Created\ncontext pre-filled from inspection/] C --> G[/Task Manually Created\noperator or inspector enters details/] D --> G E --> H[/Task Auto-Created\nlinked to inspection report/] F --> I[(Task Detail\nOpen Record)] G --> I H --> I I --> J([Task Lifecycle ↓ see Diagram 2]) style A fill:#0A1B4A,stroke:#0A1B4A,color:#fff style F fill:#fef3c7,stroke:#fcd34d,color:#92400e style G fill:#fef3c7,stroke:#fcd34d,color:#92400e style H fill:#fee2e2,stroke:#fca5a5,color:#991b1b style J fill:#fef3c7,stroke:#fcd34d,color:#92400e

Key point: Tasks created from inspections carry context automatically (location, section, item description, inspector). Tasks created manually via "New Risk" require the operator or inspector to fill in the details. Both end up in the same Tasks list.

4

Role-Based Views

What each user sees — same app, different scope

flowchart TD A([RMMInspector]) --> B{User Role} B -- Inspector / Owner --> C[/Global View\nAll locations/] B -- Operator --> D[/Unit View\nTheir restaurant only/] C --> C1[Executive Dashboard\nall locations · KPIs · alerts] C --> C2[All Inspections\nacross every unit] C --> C3[All Tasks\nwith restaurant tag] C --> C4[Reports & Analytics\nregion-wide] C --> C5[Locations List\ndrill into any unit] D --> D1[Unit Dashboard\ntheir scores · upcoming inspections] D --> D2[Their Inspections\nscheduled + completed] D --> D3[Their Tasks\nno location tag needed] D --> D4[Task Detail\nevidence + resolution] style A fill:#0A1B4A,stroke:#0A1B4A,color:#fff style B fill:#0A1B4A,stroke:#0A1B4A,color:#fff style C fill:#dbeafe,stroke:#93c5fd,color:#1e40af style D fill:#dcfce7,stroke:#86efac,color:#166534

Key point: The Operator view is scoped — they only ever see their own unit's data. No location filtering needed. The Inspector/Owner view is global with location tags on everything. Both roles share the same Task Detail screen and task lifecycle. Role distinction is built later.

Dashboard Widget Map

Key point: Both dashboards pull from the same data sources — the difference is scope (all units vs one unit). "Upcoming Inspections" is the most important widget to build next, as it anchors both the Owner and Operator experience.
